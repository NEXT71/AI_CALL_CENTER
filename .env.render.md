# Render-Specific Notes

## Local .env Files Not Used
Render uses its own environment variable management through the dashboard. The `.env` files are only for local development.

## Setting Environment Variables in Render

### Backend API Service
1. Go to https://dashboard.render.com
2. Select `ai-callcenter-backend` service
3. Go to "Environment" tab
4. Add variables:
```
MONGO_URI=mongodb+srv://Nextel:Nextel123@cluster0.9eph1nl.mongodb.net/AICallCenter
```

### AI Service
1. Select `ai-callcenter-ai-service` service
2. Go to "Environment" tab
3. Add variables:
```
HF_TOKEN=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
WHISPER_MODEL=base
ALLOWED_ORIGINS=https://your-app.vercel.app,https://ai-callcenter-backend.onrender.com
```

### Worker Service
1. Select `ai-callcenter-worker` service
2. Go to "Environment" tab
3. Add variables:
```
MONGO_URI=mongodb+srv://Nextel:Nextel123@cluster0.9eph1nl.mongodb.net/AICallCenter
```

## Automatic Service Linking
The `render.yaml` automatically links services:
- Redis host/port → Backend & Worker
- AI Service URL → Backend & Worker
- JWT secrets auto-generated

## Vercel Environment Variables
1. Go to https://vercel.com/dashboard
2. Select your project
3. Settings → Environment Variables
4. Add:
```
VITE_API_URL=https://ai-callcenter-backend.onrender.com/api
```

## Important: Update After Deployment
After Vercel gives you a URL (e.g., `https://your-app.vercel.app`):
1. Go back to Render → AI Service → Environment
2. Update `ALLOWED_ORIGINS` to include your Vercel URL
3. Click "Manual Deploy" to apply changes
